# 🤖 基于 STM32F103C8T6 的自带电池安全型智能颜色分拣系统

本项目仿照 Bilibili 博主“会学习的单片机”同款制作。系統通過數字顏色傳感器實時檢測傳送帶上通過的物料顏色，在單片機精確計算物料移動延遲後，驅動微型舵機進行物理攔截與分揀分流。

由于本项目傳送帶**自帶獨立電池盒供電**，全系統實現了**純弱電全安全架構**，接線極致簡化，徹底絕緣了任何超壓、冒煙與燒毀器件的風險，對新手開發極其友好。

---

## 🛠️ 1. 硬件架构与物料清单

本项目全部使用弱电免焊接方式搭建，核心依靠大号面包板、L298N 的螺丝端子和彩色杜邦线实现互联。

| 分类 | 元件名称 | 关键规格与选型建议 | 在项目中的核心作用 |
| :--- | :--- | :--- | :--- |
| **主控** | STM32最小系统板 | STM32F103C8T6 核心板（**LQFP48 封装**） | 整個系統的控制大腦 |
| **调试** | ST-Link 下载器 | ST-Link V2 | 用於程序的燒錄與在線 Debug 仿真 |
| **输送** | 传送带套件 | 椴木激光切割套件 + 黄色TT电机（**自带独立电池盒**） | 物料的输送载体（依靠自带电池驱动） |

## 🔌 2. 全安全弱电系统接线规范

> ⚠️ **核心电学大命令（安全共地）**：
> 1. 傳送帶由于自帶電池，**绝对不能再外接任何 9V 或 12V 适配器电源**，防止两路电源在电机内部撞击短路！
> 2. L298N 驱动板上的 **`5V EN` 黑色跳线帽必须正常插着**。
> 3. 必須拿一根杜邦線，一頭擰進 L298N 的 **`GND`** 端子（與傳送帶電池盒的**負極**死死擰在一起），另一頭插在面包板的 **负极列（-）**！這一步叫**全局安全共地**，不共地則單片機信號無法傳輸！

### 2.1 傳送帶自帶電池動力回路
1.  **自帶電池盒的正極线** ➡️ 直接擰入 L298N 綠色端子的 **`12V`**（借路進去給電機供電）。
2.  **自帶電池盒的負極线** ➡️ 直接擰入 L298N 綠色端子的 **`GND`**。
3.  **傳送帶電機兩根紅黑引線** ➡️ 直接擰入 L298N 的 **`OUT1`** 和 **`OUT2`** 端子。

### 2.2 控制與外設弱電回路
*   **主控供電**：STM32 單片機直接插著手機充電頭（或电脑 USB），從單片機引出 `5V` ➡️ 接面包板 **正极列（+）**；引出 `GND` ➡️ 接面包板 **负极列（-）**。
*   **控制信號對號入座表**：

| 外设组件 | 组件引脚名称 | 连接目标（单片机/面包板） | 对应 STM32 硬件资源位置 |
| :--- | :--- | :--- | :--- |
| **L298N驱动板** | `IN1` / `IN2` | ➡️ STM32 的 **`PA0`** / **`PA1`** | GPIO_Output（控制傳送帶前進與停止） |
| **SG90 舵机** | 信号线(橙) / 正极(红) / 负极(棕) | ➡️ STM32 的 **`PA2`** / 面包板 `+` / 面包板 `-` | **`TIM2_CH3`**（硬件獨立 PWM 輸出） |
| **TCS3200传感器**| `OUT` (核心數據方波) | ➡️ STM32 的 **`PA6`** | **`TIM3_CH1`**（硬件輸入捕獲，開啟全局中編） |
| | `S2` / `S3` (通道選擇) | ➡️ STM32 的 **`PB0`** / **`PB1`** | GPIO_Output = 輪詢切換紅綠藍濾光片 |
| | `S0` / `S1` / `VCC` / `GND` | ➡️ 面包板 `+` / `+` / `+` / `-` | 傳感器配置輸出 100% 頻率比例與供電 |

## 💻 3. 软件开发与配置说明（CubeMX 核心防错要点）

本项目採用 **STM32CubeMX + HAL 库** 開發，完美適配 **Keil 5 新版 ARM Compiler V6 编译器**。

### 3.1 STM32CubeMX 關鍵配置
1.  **SYS ➡️ Debug**：必須改選為 **`Serial Wire`**！若誤選為 `No Debug`，芯片將在開機第一納秒自殘式關閉調試大門，引發 `No target connected` 死鎖。
2.  **RCC ➡️ High Speed Clock (HSE)**：改選為 **`Crystal/Ceramic Resonator`**。右側芯片圖的 **`PD0` 和 `PD1` 會自動亮起綠燈（顯示為 RCC_OSC_IN/OUT）**，此爲正常的硬件時鐘通道物理綁定。
3.  **时钟配置（Clock Configuration）**：在大主干道 **`HCLK (MHz)`** 輸入框裡重新敲入 **`72`** 敲回車！全盤滿血 **72MHz 穩定運行**。
4.  **优化等级回退（防延时死循环）**：進入 Keil 5 點擊魔術棒 ➡️ **`C/C++ (AC6)`** ➡️ 將 **`Optimization`** 必須死死修改為 **`-O0`（不优化）**！若開啟高級優化，編譯器會自作聰明地將系統滴答計數器（SysTick）優化進緩存，導致 `HAL_Delay()` 陷入永久的死循環卡死。
5.  **调试时序修改**：點擊魔術棒 ➡️ **`Debug`** ➡️ 右側 **`Settings`** ➡️ 左下角將 **`Connect:` 改選為 `under Reset`**，**`Reset:` 改選為 `SYSRESETREQ`**。

## 📝 4. 源码核心架构与“一脚油门踩死”极简代码

### 4.1 主程序開火配置 (`main.c`)
在 `main()` 函數的 **`/* USER CODE BEGIN 2 */`** 安全區內，必須手動補上定時器的開火鑰匙，否則外設不工作：
```c
/* USER CODE BEGIN 2 */
HAL_TIM_PWM_Start(&htim2, TIM_CHANNEL_3);   // 啟動舵機 PWM 引擎
HAL_TIM_IC_Start_IT(&htim3, TIM_CHANNEL_1); // 啟動顏色傳感器輸入捕獲大閘
Set_Servo_Angle(90);                       // 舵機初始化歸位
HAL_Delay(500); 
/* USER CODE END 2 */
```

### 4.2 核心业务代码 (`while(1)`)
由於自帶電池電壓安全平穩，**我們不需要編寫任何複雜的 PWM 脈衝限速代碼**。直接一腳油門踩死，讓傳送帶平穩順滑地全速前進：
```c
while (1)
{
    // 🚂 傳送帶一腳油門踩死，自帶電池安全供電，平穩前進不發燙
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_0, GPIO_PIN_SET);   
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_1, GPIO_PIN_RESET); 

    // 🔴 輪詢切換到“红色通道”讀取數據
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_RESET); 
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_1, GPIO_PIN_RESET); 
    Color_Frequency_Count = 0;                            
    HAL_Delay(100); 
    Red_Val = Color_Frequency_Count;                      

    // 🟢 輪詢切換到“绿色通道”讀取數據
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_SET);   
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_1, GPIO_PIN_SET);   
    Color_Frequency_Count = 0;                            
    HAL_Delay(100); 
    Green_Val = Color_Frequency_Count;                    

    // ⚖️ 核心顏色判定與雙向舵機撥料分揀
    if (Red_Val > (Green_Val + 50)) {
        HAL_Delay(1000); Set_Servo_Angle(0); HAL_Delay(400); Set_Servo_Angle(90); HAL_Delay(1000); 
    }
    else if (Green_Val > (Red_Val + 50)) {
        HAL_Delay(1000); Set_Servo_Angle(180); HAL_Delay(400); Set_Servo_Angle(90); HAL_Delay(1000); 
    }
}
```
