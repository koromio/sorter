# 🤖 基于 STM32 的颜色识别传送带智能分拣系统

本项目仿照 Bilibili 博主“会学习的单片机”同款制作。系统通过数字颜色传感器实时检测传送带上通过的物料颜色，在单片机精确计算物料移动延迟后，驱动微型舵机进行物理拦截与分拣分流，模拟工业自动化流水线。

---

## 🛠️ 1. 硬件架构与物料清单

本项目全部使用免焊接方式搭建，核心依靠大号面包板、L298N的螺丝端子和彩色杜邦线实现互联。

| 分类 | 元件名称 | 关键规格与选型建议 | 在项目中的核心作用 |
| :--- | :--- | :--- | :--- |
| **主控** | STM32最小系统板 | STM32F103C8T6 核心板 | 整个系统的控制大脑 |
| **调试** | ST-Link 下载器 | ST-Link V2 | 用于程序的烧录与在线调试 |
| **架构** | 传送带套件 | 椴木激光切割套件 + 黄色TT直流减速电机 | 物料的输送载体 |
| **供电** | 直流电源适配器 | **9V 2A 大圆头(5.5x2.1mm)** 适配器 | 全系统的独立动力源（拒绝用电脑USB/ST-Link直接带电机） |
| **驱动** | 电机驱动模块 | 经典红板 **L298N**（带大散热片） | 消化大电流，驱动传送带；降压出 5V 供给单片机和外设 |

## 🔌 2. 全系统安全接线规范

> ⚠️ **致命禁区**：电源适配器 9V 线和传送带电机线**绝对不能用杜邦线**，必须使用剥线裸线直接塞进 L298N 螺丝端子并拧紧！

### 2.1 动力与电源大电流区（不进面包板）
*   **9V电源适配器**：正极（红/白虚线） ➡️ L298N 的 **`12V`** 端子；负极（纯黑） ➡️ L298N 的 **`GND`** 端子。
*   **传送带电机**：两根红黑引线 ➡️ 直接拧入 L298N 的 **`OUT1`** 和 **`OUT2`** 端子。

### 2.2 信号与弱电低电流区（通过面包板分流）
*   **全系统共地与分流**：
    *   从 L298N 的 **`5V`** 输出端引出一根公对公杜邦线 ➡️ 插在面包板的**正极列（+）**。
    *   从 L298N 的 **`GND`** 端（与电源负极合并拧紧）引出一根公对公杜邦线 ➡️ 插在面包板的**负极列（-）**。
*   **各个外设对号入座接线表**：

| 外设组件 | 组件引脚名称 | 连接目标（单片机/面包板） | 对应 STM32 硬件资源位置 |
| :--- | :--- | :--- | :--- |
| **STM32核心板** | `5V` / `GND` | ➡️ 面包板 `+` / 面包板 `-` | 系统供电与全局共地（板载LDO自动降为3.3V） |
| **L298N驱动板** | `IN1` / `IN2` | ➡️ STM32 的 **`PA0`** / **`PA1`** | GPIO_Output（高低电平控制电机启停与正反转） |
| **SG90 舵机** | 信号线(橙) / 正极(红) / 负极(棕) | ➡️ STM32 的 **`PA2`** / 面包板 `+` / 面包板 `-` | **`TIM2_CH3`**（硬件独立 PWM 输出，50Hz 脉冲控制） |
| **TCS3200传感器**| `OUT` (核心数据方波) | ➡️ STM32 的 **`PA6`** | **`TIM3_CH1`**（硬件输入捕获模式，开启全局中断） |
| | `S2` / `S3` (通道选择) | ➡️ STM32 的 **`PB0`** / **`PB1`** | GPIO_Output（单片机输出电平轮询切换红绿蓝滤光片） |
| | `S0` / `S1` (输出比例) | ➡️ 全部直接插在单片机/面包板 **`3.3V / 5V`** | 始终给高电平，配置传感器输出 100% 频率比例 |
| | `VCC` / `GND` / `LED` | ➡️ 面包板 `+` / 面包板 `-` / 悬空不接 | 传感器供电与白色补光灯常亮控制 |

## 💻 3. 软件开发与配置说明（CubeMX关键要点）

本项目采用 **STM32CubeMX + HAL 库** 的图形化架构进行开发，完美支持 **Keil 5 新版 ARM Compiler V6 编译器**。

### 3.1 STM32CubeMX 避坑配置参数
1.  **System Core ➡️ SYS ➡️ Debug**：配置为 **`Serial Wire`**（确保 ST-Link 能正常下载通信）。
2.  **Timers ➡️ TIM2 (舵机控制)**：
    *   `Channel3` 设置为 `PWM Generation CH3`（生成控制 PA2 引脚）。
    *   参数配置：`Prescaler (PSC)` = **`71`**；`Counter Period (ARR)` = **`19999`**。
3.  **Timers ➡️ TIM3 (颜色频率捕获)**：
    *   `Clock Source` 必须勾选 **`Internal Clock`**（提供内部时钟源心跳，否则定时器无法运转）。
    *   `Channel1` 设置为 **`Input Capture direct mode`**。
    *   参数配置：`Prescaler` = **`71`**；`Counter Period` = **`65535`**。
    *   右侧点击 **`NVIC Settings`** 标签页，必须勾选 **`TIM3 global interrupt` 处的 Enabled 开关**（开启全局中断用于脉冲自增计数）。
4.  **GPIO 冲突提示（重点说明）**：
    *   当 `PB0`, `PB1` 被手动配置为 `GPIO_Output` 后，左侧 `TIM3` 可能会亮起黄色感叹号。此现象属于正常的功能复用提示，非配置错误，不影响 `TIM3_CH1` 正常工作，直接点击 **`GENERATE CODE`** 即可。

### 3.2 编译器环境切换（MDK V6 修复）
若编译时提示 `ARM-Compiler 'Default Compiler Version 5' which is not available`，请通过以下方式一键切换：
*   点击 **Options for Target（魔术棒）** ➡️ **Target** 选项卡 ➡️ **ARM Compiler** ➡️ 修改为 **`Use default compiler version 6`**。

## 📝 4. 核心源码逻辑与大括号防错

### 4.1 中断计数 (`stm32f1xx_it.c`)
在 `TIM3_IRQHandler` 中，使用两个下划线开头的 HAL 库底层宏，在每次颜色方波跳变时进行自增：
```c
if (__HAL_TIM_GET_FLAG(&htim3, TIM_FLAG_CC1) != RESET) {
    __HAL_TIM_CLEAR_IT(&htim3, TIM_IT_CC1);
    Color_Frequency_Count++;
}
```

### 4.2 双向对称分拣算法 (`main.c`)
*   **物理校准**：将舵机开机初始化代码设置为 **`1500`（对应 90 度正中间位置）**。通电定格后，再将雪糕棒机械臂**垂直**插入轴心，以此实现完美的“左拨-复位-右拨”对称分拣动作。
*   **语法规范**：所有 `HAL_GPIO_WritePin` 中的参数必须使用**纯英文半角逗号 `,`** 以及**英文分号 `;`** 结尾，且 `HAL_Delay` 严格区分大小写。
*   **嵌套防错**：自定义的 `Set_Servo_Angle()` 必须放在 `main()` 函数外部（USER CODE 0 区域），且 `main()` 函数尾部必须有完整的双重右大括号 `}` 来闭合 `while(1)` 和 `main` 本身，防止函数嵌套报错。

## 🚀 5. 快速启动与现场联调

1.  **环境检查**：点击 Keil 5 左上角的双向下箭头进行 Rebuild 全编译，确保控制台输出 `"xxxx.axf" - 0 Error(s), 0 Warning(s)`。
2.  **物理架设**：用打火机烧热熔胶棒将颜色传感器固定在皮带前半段上方（保持 1~1.5cm 距离）。
3.  **避光防噪**：由于 TCS3200 属于光电频率传感器，极易受到房间白炽灯和环境光波动的影响。建议用硬纸盒或 3D 打印件为传感器做一个**黑色的简易遮光罩**，项目识别准确率可直接提升 200%！
